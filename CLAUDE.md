# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

**Mee** — a Vietnamese-first AI meeting agent. Pipeline: paste text / upload audio / live record → transcript → WYSIWYG "Clean" editor → per-recording **Biên bản phiên họp (MoM)** + aggregated **project summary**. Backend is FastAPI + LangGraph; LLM is Qwen3 (OpenAI-compatible), STT is PhoWhisper/Whisper, embedding is bge-m3 over Postgres+pgvector.

The README.md is extensive and written in Vietnamese — read it for user-facing flows and DB migration history.

## Commands

The active virtualenv is `venv/` (not `.venv/`, despite some README/script references). Use `venv/bin/python`, `venv/bin/pip`, `venv/bin/alembic`.

```bash
# Install
venv/bin/pip install -r requirements.txt

# Apply DB migrations (required before first run — pgvector extension must exist)
venv/bin/alembic upgrade head
venv/bin/alembic revision --autogenerate -m "description"   # new migration

# Run backend — starts BOTH servers: HTTP :8001 and WebSocket :9091
venv/bin/python run_meeting.py

# React frontend (recommended) — Vite dev server :5173, proxies /api→:8001, /ws→:9091
cd meeting_frontend_react && npm install && npm run dev
cd meeting_frontend_react && npm run build      # production → dist/

# Local Postgres+pgvector + Adminer (only if not using a remote DATABASE_URL)
docker compose --profile local up -d            # postgres:5435, adminer:8080

# One-shot maintenance scripts
venv/bin/python scripts/backfill_embeddings.py  # embed existing memory_events
venv/bin/python scripts/regenerate_mom.py <transcript.txt>
```

`main.py` is a minimal alternative entrypoint (HTTP only, port 8080) — `run_meeting.py` is the real one.

### Tests
The files in `tests/` (`test_server.py`, `test_client.py`, `test_vad.py`, `test_batch_inference.py`) test the **legacy `whisper_live` streaming system**, not the `meeting/` backend — they import `whisper_live.server`, `jiwer`, and `whisper`. There is currently **no test suite for the `meeting/` package**. Don't assume `pytest` exercises the FastAPI app.

## Architecture

### Two-level data model (the core mental model)
```
meetings        = a Project  (title, attendees, is_pinned, project_summary_json)
  └─ recordings = a Phiên/session  (session_label, started_at, mom_json, clean_segments)
       └─ transcript_segments  (per-sentence: seq, original_text, edited_text)
```
- **MoM is per-recording**, stored in `recordings.mom_json` (JSONB). Generated by `meeting/graphs/mom_graph.py`.
- **Project summary** aggregates all of a project's recording MoMs by `started_at` into a timeline + narrative, stored in `meetings.project_summary_json`. Generated by `meeting/services/project_summarizer.py`.
- **Clean transcript**: TipTap editor output lives in `recordings.clean_segments` (JSONB with `edited_html` / `edited_text`). MoM generation **prefers `edited_text` over raw transcript** when present.

### Backend layout (`meeting/`)
- `app.py` — FastAPI factory (`create_app`). Holds legacy in-memory session endpoints, `/api/transcribe` (Whisper file upload with auto-chunking), vocab-pool endpoints, and mounts the legacy `meeting_frontend/` SPA at `/`. Lifespan inits/closes the LangGraph checkpointer.
- `api/meetings.py` — the main DB-backed REST router (meetings, recordings, segments, clean, MoM, project summary).
- `api/chat.py` — Chat HITL router (sessions, messages, pending-actions approve/reject).
- `graphs/mom_graph.py` — LangGraph `StateGraph`: `load_transcript → read_memory → generate_mom → save_results`. Compiled with `AsyncPostgresSaver`; **`thread_id = recording_id`**, so re-running the same recording resumes from the failed node.
- `graphs/chat_graph.py` — chat agent graph (classify intent → propose action → HITL).
- `graphs/checkpointer.py` — global singleton `AsyncPostgresSaver`. Creates `checkpoints` / `checkpoint_writes` / `checkpoint_blobs` tables on `setup()`.
- `services/` — `memory_service.py` (hybrid retrieval: keyword tsvector + bge-m3 vector + RRF fusion + optional rerank), `embedding.py`, `reranker.py`, `transcript_cleaner.py`, `project_summarizer.py`.
- `note_generator.py` — the MoM LLM call. **Map-reduce** for long transcripts (thresholds tuned for 8K-context Qwen3-8B: `SINGLE_CALL_THRESHOLD=10k`, `CHUNK_SIZE=9k`, `MAX_TRANSCRIPT_CHARS=60k`). Strips Qwen3 `<think>` tags. Classifies content into action_items / decisions / commitments / blockers.
- `report_generator.py` — MoM JSON → Markdown (`./output/MoM_*.md`).
- `db/base.py` (async engine/session), `db/models.py` (ORM), `db/repositories.py` (queries).

### Frontends (two of them)
- `meeting_frontend_react/` — React 18 + TS + Vite + TipTap. **The recommended one.** State in `src/store/AppContext.tsx`, typed API in `src/api/client.ts`, live recording over WebSocket in `src/hooks/useLiveRecording.ts`, i18n VI/EN in `src/i18n.ts`.
- `meeting_frontend/` — legacy vanilla JS SPA, served directly by FastAPI at `http://localhost:8001/`. Kept as fallback.

### External AI services (all OpenAI-compatible, configured via `.env`)
- **STT**: VNG MaaS Whisper, OR self-hosted PhoWhisper+pyannote (`tools/phowhisper-server/`, deploy to an L40 GPU).
- **LLM**: self-hosted Qwen3-8B via vLLM, OR VNG MaaS.
- **Embedding**: bge-m3 (1024-dim) via VNG MaaS.

## Critical gotchas

- **One DATABASE_URL, three drivers.** The same Postgres URL is normalized three ways: SQLAlchemy app → **asyncpg** (`db/base.py`); Alembic → **psycopg2** sync via `DATABASE_URL_SYNC` or derived (`alembic/env.py`); LangGraph checkpointer → **psycopg3** with `+asyncpg` stripped (`graphs/checkpointer.py`). When changing connection logic, fix all three.

- **Embedding env var name mismatch.** `meeting/services/embedding.py` reads `EMBED_BASE_URL` / `EMBED_API_KEY` / `EMBED_MODEL` / `EMBED_DIM`, but `.env.example` documents `EMBEDDING_BASE_URL` / `EMBEDDING_API_KEY` / `EMBEDDING_MODEL`. The `.env` must set the **`EMBED_*`** names for memory/embedding to work. Don't trust `.env.example` here.

- **dotenv is loaded with `override=True, interpolate=False`.** This is deliberate so passwords containing `$` survive. Keep that when adding env loading.

- **OpenAI clients are created per-service, not shared.** Each of `note_generator`, `transcript_cleaner`, `reranker`, `project_summarizer`, `chat_graph`, `embedding`, `vocab_store` constructs its own `OpenAI(...)`. There is no shared client singleton.

- **Whisper handling lives in `app.py`**: known-hallucination filtering (`WHISPER_HALLUCINATIONS`), and auto-chunking of audio >24MB into 10-min 16kHz mono WAV chunks (`WHISPER_FILE_SIZE_LIMIT`, `CHUNK_DURATION_SEC`). The Whisper `initial_prompt` is engineered to anchor Vietnamese + preserve English tech terms (`_build_whisper_prompt`).

- **pgvector is required**, not optional. `memory_events.embedding` is `vector(1024)` with an IVFFlat index (migration `0006`). The DB must have the extension before `alembic upgrade head`.
